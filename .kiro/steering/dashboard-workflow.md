# Dashboard Development Workflow

## Scope & Commits

**Important:** All changes must be made **only in the `/dashboard` folder**.

- ✅ Commit and push changes made in `/dashboard`
- ❌ Do NOT commit, push, or create PRs for changes outside `/dashboard`
- ❌ Ignore any files/folders created outside `/dashboard`

## Branch & PR Workflow

### When to Create a Branch
Create a new branch when:
- Implementing a new feature
- Making significant edits
- Making any substantial changes to `/dashboard`

### After Completing Work
1. Ask the user: "Do you want me to create a pull request or commit?"
2. If user says "commit", provide:
   - **PR Title**: Best summary of all changes (concise, descriptive)
   - **PR Description** including:
     - Summary of what was done
     - Benefits for users and the app
     - Technical details if needed
     - Thanks at the end

### PR Title Guidelines
- Summarize all changes concisely
- If multiple changes, create a comprehensive summary
- Keep it clear and descriptive

### PR Description Guidelines
- Start with a summary of what was done
- Explain benefits for users and the app
- Include technical details if relevant
- End with a thank you message

## Workflow Summary

1. **New Request** → Check if branch needed
2. **Make Changes** → Only in `/dashboard`
3. **Complete Work** → Ask about PR or commit
4. **Provide Details** → Title and description ready for user to create PR
